=>Γενικη μορφη της εντολης για διεξαγωγη της τυχαιας δειγματοληψιας με αντικατασταση.
H εντολη boot ειναι η βασικη εντολη της μεθοδου:μεσω αυτης προσδιοριζεται το συνολο στο οποιο
θελουμε να γινει η τυχαια δειγματοληψια, καθως και το function που θελουμε να εφαρμοσουμε σε αυτο το συνολο.  
Το function που εφαρμοστηκε στη δικη μας περιπτωση ηταν η καταμετρηση της εμφανισης καθε αμινοξικης αλλαγης.
Η σειρα των γεγονοτων εχει ως εξης:
α)προσδιοριζεις το συνολο στο οποιο εργαζεσαι (στην περιπτωση μας ειναι το συνολο των snps και συγκεκριμενα η πληροφορια που αφορα στην αμινοξικη
αλλαγη π.χ Α->D)
β)στη συνεχεια προσδιοριζεις το function που θες να εφαρμοσεις πανω στο συνολο (εδω ειναι η καταμετρηση, π.χ Α->D: 15 φορες)
γ)επειτα η εντολη boot αυτο που θα κανει ειναι να παρει το αρχικο σου συνολο και τυχαια να στο αναδιαμορφωση, φτιαχνοντας ενα θεωρητικο
συνολο και η διαδικασια αυτη επαναλαμβανεται για τουλαχιστον 100 φορες (μεσω της εντολης replicate), ωστε να μειωθει ο παραγοντας της τυχης.
(ετσι π.χ στα θεωρητικα συνολα μπορει να εχεις Α->D:13, A->D:14, A->D:17 κτλ).
δ)απο τα τυχαια θεωρητικα συνολα υπολογιζεις το μεσο ορο των τιμων που σου δινει το function (δλδ η καταμετρηση εδω) καθως και την τυπικη αποκλιση
για καθε συνολο τιμων στα θεωρητικα συνολα. (Το βημα αυτο στη δικη μας περιπτωση λογω της ιδιομορφης μορφολογιας των δεδομενων διεξηχθηκε με scripts
που γραφτηκαν στη γλωσσα προγραμματισμου perl).
